En resumen, la notaci√≥n siempre es de la siguiente forma: \[ \text{Base.de.datos}[\underbrace{f}_{fila},\overbrace{c}^{columna}] \] Esta notaci√≥n notaci√≥n matricial est√°ndar en el mundo de computaci√≥n y de las matem√°ticas. Siempre siempre primero es fila luego columna. ¬°No lo olvides pues nos ser√° √∫til para los modelos!

Otra forma de seleccionar una columna es con $ seguido del nombre de la columna. Por ejemplo:

mortalidad.2017$edo_civil
## [1] 3 5 3 3 2 5

Por ciertos errores en los que se pueden incurrir al crear funciones, se recomienda que se utilice siempre la notaci√≥n de doble corchete y se evite el signo de $. Pero ambos m√©todos funcionan. S√≥lo recuerda que la notaci√≥n es as√≠:

\[ \text{Base.de.datos}\$\text{Columna}[\underbrace{f}_{\text{Fila}}] \]

Para seleccionar m√∫ltiples filas o columnas, como vimos en el ejemplo anterior, hay que crear un vector usando c2 Se llama c por concatenate (concatenar).. Un vector es una lista ordenada de variables del mismo tipo. Por ejemplo:

mi.vector <- c(1, 141, 12)

Es un vector de 3 entradas. De hecho, la forma de acceder a sus entradas es la misma que la de los data.frames con un agregado: ¬°s√≥lo hay filas, no hay columnas!

Intuitivamente podemos pensar un vector como una columna de una base de datos. En ella todas las variables son del mismo tipo.

Para acceder a las entradas de un vector es de la siguiente forma:

#Para un vector s√≥lo pongo el n√∫mero de entrada ¬°no hay columna!
mi.vector[2] #entrada 2
## [1] 141

En general la notaci√≥n es la siguiente: \[ \text{vector}[\underbrace{i}_{\text{Entrada}}] \]

Un vector que ya nos encontramos antes es el de columnas. Podemos, por ejemplo, ver el nombre de la cuarta columna de la base combinando lo que sabemos de vectores con el comando colnames:

#Regresa el nombre de la 5a columna
colnames(mortalidad.2017)[5]
## [1] "tloc_resid"

La pregunta tambi√©n puede hacerse al rev√©s: podemos pedirle a R que nos conteste cu√°l es el n√∫mero de columna para una columna dada. Por ejemplo, ¬øcu√°l es el n√∫mero de columna para edad?

Funciones en R

Una funci√≥n la podemos pensar como un pedazo de c√≥digo que transforma un poco de c√≥digo en otra cosa.

Una funci√≥n toma ciertos valores de input y regresa ciertos outputs

Por si te lo preguntabas, s√≠, tambi√©n el diagrama lo hicimos con ggplot2 ;)

Por ejemplo la siguiente funci√≥n toma como Input un n√∫mero y le suma 5 (yo s√© que es una funci√≥n muy in√∫til pero vamos paso a paso):

#Defino mi funci√≥n
suma.5 <- function(mi.numero){
  return(mi.numero + 5)
}

#Pruebo mi funci√≥n
suma.5(10)
## [1] 15
suma.5(1)
## [1] 6
suma.5("Ping√ºino")
## Error in mi.numero + 5: non-numeric argument to binary operator

La estructura b√°sica de una funci√≥n en R es como sigue: \[ \begin{aligned} \text{Nombre.de.la.function}& \leftarrow\textrm{function}(\overbrace{\textrm{Inputs}}^{\text{Lo que necesita para funcionar}})\{ \\ & \text{C√≥digo} \\ & \text{return}(\underbrace{\text{Output}}_{\textrm{Lo que resulta}}) \\ \} \end{aligned} \] Veamos otra funci√≥n:

#Funci√≥n que dice ¬°Hola ----! donde ---- es el nommbre que se mete
mi.saludo <- function(nombre){
  saludo <- paste0("¬°Hola ", nombre, "!")
  return(saludo)
}

#Ejemplos

## [1] "¬°Hola Ana!"
mi.saludo("Jes√∫s")
## [1] "¬°Hola Jes√∫s!"

Una funci√≥n puede tener m√∫ltiples par√°metros, por ejemplo la funci√≥n que resta un n√∫mero a otro:

resta.numeros <- function(num.1, num.2){

Por √∫ltimo, una funci√≥n tambi√©n puede regresar m√∫ltiples valores. Para ello est√° el comando list. Por ejemplo, la funci√≥n que regresa la media y desviaci√≥n est√°ndar de un vector:

media.y.varianza <- function(mi.vector){
  #' media.y.varianza
  #'
  #' Funci√≥n que calcula la media y desviaci√≥n est√°ndar de un vector
  #' @param mi.vector Un vector de R al cual calcular media y desviaci√≥n
  #' @return mi.lista Lista de par√°metros que contiene:
  #'           media    Media del vector
  #'           desv.est Desviaci√≥n est√°ndar del mismo
  
  #Calculamos la media
  media    <- mean(mi.vector)
  
  #Calculamos la desviaci√≥n est√°ndar
  desv.est <- sd(mi.vector)
  
  return(list(promedio = media, desviacion = desv.est))
}

#Veamos un ejemplo:
un.vector <- c(0.27, -0.34, 0.11, -0.6, 0.82, -0.41)

#Podemos calcular ambas

## $promedio
## [1] -0.025
## 
## $desviacion
## [1] 0.5286492
#Para acceder a un valor hay que usar $
media.y.varianza(un.vector)$promedio
## [1] -0.025

donde mean es una funci√≥n de R que calcula la media de un vector:

#Cremos un vector
x <- c(1, 0.5, 2, 0.6, 0.4, 0, 0.2)

## [1] 0.6714286

y sd calcula su desviaci√≥n est√°ndar:

analisis.mortalidad <- function(base, anio, causas){
  
  # 1 Leemos la base
  mortalidad <- read.csv(base)
  
  # 2 Elegimos s√≥lo las variables que nos interesan
  mortalidad <- mortalidad %>%
    select(c("ent_regis", "sexo", "edad", 
             "dia_ocurr", "mes_ocurr", "anio_ocur", "causa_def"))
  
  #Cambiamos los nombres de las variables
  colnames(mortalidad) <- c("Entidad", "Sexo", "Edad", 
                                         "D√≠a", "Mes", "A√±o", "Causa") 
  
  #Aqu√≠ cambiamos lo de sexo
  mortalidad <- 
    indicadores.mortalidad(mortalidad, dic.sexo, 
                           "Sexo", "CVE", "DESCRIP")
  
  #Aqu√≠ cambiamos lo de causa
  mortalidad <- 
    indicadores.mortalidad(mortalidad, dic.causa, 
                           "Causa", "CVE", "DESCRIP")
  
  #Aqu√≠ cambiamos lo de estado
  mortalidad <- 
    indicadores.mortalidad(mortalidad, dic.entidad, 
                           "Entidad", "cve_ent", "nom_loc")
  
  # 4 Filtramos
  mortalidad.causas <- mortalidad %>% 
    filter(Causa %in% !!causas) %>%
    count(Sexo, Mes, Entidad)
  
  # 5 Agregar√© una columna a la base que se llame a√±o con 2016
  mortalidad.causas[, "A√±o"] <- anio
  
  # Renombraremos la columna n como defunciones
  mortalidad.causas <- mortalidad.causas %>% rename("Defunciones" = "n")

  return(mortalidad.causas)
}
